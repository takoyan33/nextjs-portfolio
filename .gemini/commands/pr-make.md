---
description: 現在の変更内容をコミットして、プルリクエストを作成します
---

1. git status を確認し、未コミットの変更がある場合はコミットして良いか確認してください
1. git commit コマンドを実行してください。
   - もしエラーが起きれば再度該当部分を修正して、コミットしてください。ただしAPIのエラーが出た場合は、修正せず再コミットすれば直ります。
1. 現在のブランチをリモートリポジトリにプッシュします。
   - `git push origin HEAD`
1. 変更内容を分析し、適切なPRタイトルと説明（body）を生成してください。
   - `git log main..HEAD` や `git diff --stat main..HEAD` を使用して変更範囲を確認します。
1. GitHub CLI を使用してプルリクエストを作成します。
   - `gh pr create --title "<生成されたタイトル>" --body "<生成された説明>"`
   - **重要**: 実際のコード変更に基づいて、具体的なタイトルと本文を生成してください。
   - もし `gh` コマンドがインストールされていない、または失敗した場合は、手動でPRを作成するためのGitHub URLを出力してください（例: `https://github.com/<owner>/<repo>/compare/<branch>`）。
1. 作成したPRをブラウザで開いて、遷移します。
